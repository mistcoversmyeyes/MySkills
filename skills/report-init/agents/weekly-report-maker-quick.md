# Weekly Report Maker — Quick Mode

生成本周周报草稿。快速模式下，自主汇总本周所有日报和项目进展后直接生成周报草稿，不进行交互式提问。

## Step 1：收集上下文

自主执行以下信息收集（不需要询问用户）：

1. **汇总本周日报**：读取 `report/` 目录下本周所有日报文件，提取每日完成的任务、遇到的问题、Q&A 记录
2. **获取本周 git 活动**：运行 `git log --oneline --since="last monday" --author` 查看本周所有提交记录
3. **检查项目任务追踪**：如存在 `project-scope.md`、WBS 文档或 TODO 文件，读取当前阶段任务的完成进度
4. **检查最新日报的后续计划**：作为下周计划的输入来源

## Step 2：直接生成周报草稿

基于收集到的上下文，按照 [references/weekly-template.md](../references/weekly-template.md) 中的模板规范直接生成周报草稿。

对于无法确认的信息，做合理推测并用 `<!-- TODO: 确认 -->` 标记，便于用户在确认环节修正。

关键要求：

- 标题格式：`# 第N周 周报`
- 按工作方向（Scope）组织本周成果，每个 Scope 下概述关键进展
- 从日报细节条目中提炼出成果级概述（不是简单罗列日报内容）
- 问题与复盘段落：汇总本周遇到的问题、解决方案、经验教训
- 末尾必须包含「下周计划」部分，延续最新日报的后续计划并归纳优先级

生成完毕后，将草稿返回给 SKILL.md 的确认循环环节。
