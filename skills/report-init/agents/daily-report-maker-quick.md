# Daily Report Maker — Quick Mode

生成今日日报草稿。快速模式下，自主收集项目上下文后直接生成草稿，不进行交互式提问。

## Step 1：收集上下文

自主执行以下信息收集（不需要询问用户）：

1. **定位昨日日报**：查看 `report/` 目录下最新的日报文件，读取其「后续计划」部分
2. **获取 git 近期活动**：运行 `git log --oneline --since="yesterday" --author` 查看昨日至今的提交记录
3. **扫描当前关注点**：查看最近修改的文件（`git diff --stat HEAD~5`），推测当前工作方向
4. **检查项目任务追踪**：如存在 `project-scope.md`、WBS 文档或 TODO 文件，读取其中当前阶段的任务状态

## Step 2：直接生成日报草稿

基于收集到的上下文，按照 [references/daily-template.md](../references/daily-template.md) 中的模板规范直接生成日报草稿。

对于无法确认的信息，做合理推测并用 `<!-- TODO: 确认 -->` 标记，便于用户在确认环节修正。

关键要求：

- 标题格式：`# M.DD 日报`
- 按工作方向（Scope）组织内容，每个 Scope 下列出具体任务条目
- 任务条目使用 checkbox 格式：`- [x]` 已完成，`- [ ]` 未完成/推测中
- 昨日日报「后续计划」中的条目 → 转化为今日的任务条目（状态根据 git log 推断）
- 如有问题记录线索，放入 Problem Faced 段落
- 末尾必须包含「后续计划」部分，延续昨日未完成的计划

生成完毕后，将草稿返回给 SKILL.md 的确认循环环节。
