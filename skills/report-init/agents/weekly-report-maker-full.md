# Weekly Report Maker — Full Mode

生成本周周报草稿。完整模式下，先自主汇总本周所有日报和项目进展，向用户回告已掌握的信息，再通过交互式提问补齐遗漏，最后生成周报草稿。

## Step 1：收集上下文

自主执行以下信息收集（不需要询问用户）：

1. **汇总本周日报**：读取 `report/` 目录下本周所有日报文件，提取每日完成的任务、遇到的问题、Q&A 记录
2. **获取本周 git 活动**：运行 `git log --oneline --since="last monday" --author` 查看本周所有提交记录
3. **检查项目任务追踪**：如存在 `project-scope.md`、WBS 文档或 TODO 文件，读取当前阶段任务的完成进度
4. **检查最新日报的后续计划**：作为下周计划的输入来源

## Step 2：回告已掌握信息

将收集到的信息整理后向用户报告，格式如下：

---

**本周工作概览（来自日报汇总）：**

- 本周主要工作方向：...
- 已完成的关键任务：...
- 遇到的问题及解决情况：...
- 重要的 Q&A 记录：...

**需要你补充的：**
- 有哪些工作成果日报中没有记录？
- 本周有没有关键决策或方向调整？
- 下周计划有没有需要补充或调整的？

---

## Step 3：交互式补充

根据用户的回答，持续追问直到获得足够的信息来生成高质量周报。追问方向包括但不限于：

- 日报中遗漏的工作内容
- 任务成果的总结性描述（从细节条目到成果概述）
- 本周学到的关键经验教训
- 下周计划的优先级排序

当信息充足时，告知用户"信息已够，开始生成草稿"。

## Step 4：生成周报草稿

按照 [references/weekly-template.md](../references/weekly-template.md) 中的模板规范生成周报草稿。

关键要求：

- 标题格式：`# 第N周 周报`
- 按工作方向（Scope）组织本周成果，每个 Scope 下概述关键进展
- 问题与复盘段落：汇总本周遇到的问题、解决方案、经验教训
- 末尾必须包含「下周计划」部分，按优先级列出下周工作方向和具体任务

生成完毕后，将草稿返回给 SKILL.md 的确认循环环节。
