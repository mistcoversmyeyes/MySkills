# Daily Report Maker — Full Mode

生成今日日报草稿。完整模式下，先自主收集项目上下文，向用户回告已掌握的信息，再通过交互式提问补齐关键背景，最后生成日报草稿。

## Step 1：收集上下文

自主执行以下信息收集（不需要询问用户）：

1. **定位昨日日报**：查看 `report/` 目录下最新的日报文件，读取其「后续计划」部分
2. **获取 git 近期活动**：运行 `git log --oneline --since="yesterday" --author` 查看昨日至今的提交记录
3. **扫描当前关注点**：查看最近修改的文件（`git diff --stat HEAD~5`），推测当前工作方向
4. **检查项目任务追踪**：如存在 `project-scope.md`、WBS 文档或 TODO 文件，读取其中当前阶段的任务状态

## Step 2：回告已掌握信息

将收集到的信息整理后向用户报告，格式如下：

---

**我从项目上下文中了解到的信息：**

- 昨日计划（来自上一份日报的后续计划）：...
- 近期代码活动：...
- 当前工作方向推测：...
- 项目任务状态：...

**需要你补充的：**
- 今天实际打算做什么？（与昨日计划有差异的部分）
- 有没有遇到的问题或阻塞？
- 有没有需要记录的 Q&A？

---

## Step 3：交互式补充

根据用户的回答，持续追问直到获得足够的信息来生成高质量日报。追问方向包括但不限于：

- 任务的完成状态（已完成 / 进行中 / 未开始）
- 遇到的问题及其解决方案（或待解决）
- 需要记录的问答（来自与同事/导师的交流）
- 对后续计划的调整

当信息充足时，告知用户"信息已够，开始生成草稿"。

## Step 4：生成日报草稿

按照 [references/daily-template.md](../references/daily-template.md) 中的模板规范生成日报草稿。

关键要求：

- 标题格式：`# M.DD 日报`
- 按工作方向（Scope）组织内容，每个 Scope 下列出具体任务条目
- 任务条目使用 checkbox 格式：`- [x]` 已完成，`- [ ]` 未完成
- 如有问题记录，放入 Problem Faced / Problem Solved 段落
- 如有问答，放入 Q & A 段落
- 末尾必须包含「后续计划」部分，为明天的日报提供输入

生成完毕后，将草稿返回给 SKILL.md 的确认循环环节。
