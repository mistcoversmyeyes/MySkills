# 日报模板规范

本文件定义日报的结构和格式约定，供 Daily Report Maker 生成草稿时参考。

## 文件命名与路径

- 文件名：`M.DD 日报.md`（月份不补零，日期补零）
- 路径：`report/第X周/M.DD 日报.md`（X 为中文数字，如"第一周"、"第二周"）

## 结构

```markdown
# M.DD 日报

## Scope 名称 1

- [x] 已完成的任务描述
- [x] 已完成的任务描述（可附加简要说明）
- [ ] 未完成的任务描述

## Scope 名称 2

- [x] ...
- [ ] ...

## Problem Solved
### 问题标题
- 问题描述：...
- 问题原因：...
- 解决方案：...
- 解决步骤：
  1. ...
  2. ...

## Problem Faced
### 问题标题
- 问题描述：...
- 当前状态：...（待解决 / 已绕过 / 需要帮助）

## Q & A

### Q1

背景：

希望提供的帮助：

### A1


# 后续计划

## Scope 名称 1
- [ ] 明天/后续打算做的任务

## Scope 名称 2
- [ ] ...
```

## 格式约定

- **Scope 命名**：使用工作方向的简短描述作为二级标题，如"Benchmark 测试推进"、"Agent 基建"、"VBox 编译"
- **任务条目**：使用 checkbox 格式（`- [x]` / `- [ ]`）
- **缩进描述**：任务的补充说明缩进两格挂在任务下方
- **Problem Solved vs Problem Faced**：已解决的问题用 Solved，未解决的用 Faced；同一份日报可以只有其中一种或都有
- **Q & A**：记录与同事/导师交流中产生的问题和答案，如果答案暂未获得可留空 A 部分
- **后续计划**：必须存在，是下一份日报的主要输入来源，按 Scope 分组

## 风格指南

- 任务描述简洁明了，一行说清做了什么
- 如果任务有重要细节（如数据、结论），可以用缩进的子项列出关键信息
- 不需要写"今日"等时间前缀，标题已包含日期
- Scope 之间的顺序按重要性/工作量排列，主线工作放前面
