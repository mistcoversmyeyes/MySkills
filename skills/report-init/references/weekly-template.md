# 周报模板规范

本文件定义周报的结构和格式约定，供 Weekly Report Maker 生成草稿时参考。

## 文件命名与路径

- 文件名：`第X周 周报.md`（X 为中文数字，如"第一周"、"第二周"）
- 路径：`report/第X周/第X周 周报.md`

4. ...

## 格式约定

- **一、本周学习 / 工作完成进展**：从本周所有日报中提炼出成果级条目，使用有序列表。每条描述简洁但信息完整，体现"做了什么 + 结果/产出"
- **二、遇到的困难与解决办法**：汇总本周日报中的 Problem Solved / Problem Faced，按"背景→分析→解决办法"三段式组织。如果问题少于 3 个，保留实际数量即可
- **三、下周学习 / 工作规划**：综合最新日报的后续计划，按优先级排列为有序列表
- **四、AI 相关分享**：记录本周在 AI 工具使用方面的实践、发现或心得。如果本周无相关内容可标注"本周暂无"

## 风格指南

- 周报比日报更概括，侧重成果和影响而非过程细节
- 用数据说话：如果有测试结果、性能数据等，在进展中突出
- 下周规划要具体可执行，避免模糊的方向性描述
- 问题分析要有深度：不仅是"遇到了什么问题"，还要说清分析过程和解决思路
