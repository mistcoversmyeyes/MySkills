# 工作分解结构（WBS）：<工作方向标题>

> 关联范围说明：`./scope.md`
> 最近更新：YYYY-MM-DD
>
> 本文档把 `scope.md` 中的**交付物**进一步分解为可执行的**工作包**。
> 分解原则：
> 1. **100% 法则**：子节点完整地且仅仅覆盖父节点的工作，不多不少。
> 2. **按交付物驱动**：不是按组织结构、不是按人员。
> 3. **粒度适中**：叶子工作包工作量控制在 4 小时 – 2 人天。
> 4. **不确定即声明**：无法估时的节点标 `[SPIKE]`，先做时间盒调研。

## 编码约定

- WBS 编号采用多级点号：`1`、`1.1`、`1.1.1` …
- 状态字段：`TODO / DOING / DONE / BLOCKED / DROPPED`
- 优先级：`Must / Should / Could`（与 scope.md 中的交付物优先级对齐）
